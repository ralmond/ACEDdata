Interpreting StatShop Model Descriptions

StatShop models are described in HTML using two types of files. The Assessment Model Description (suffix amd.xml) corresponds to the Portal Assembly Model. It lists which Proficiency Model(s) are used, how scores are calculated, and which Evidence/Link Models are used to score each task. The Graphical Model (suffix gm.xml) file stores the graph and probabilities for an individual Student, Evidence or Link Model. Generally speaking these files reside in the same directory, with the amd file pointing to the gm files it needs in the same directory.

Assessment Model Description

The Assessment Model Description report is divided into three sections. The first section describes the Proficiency Model, the second the Evidence and Link Models and the third describes rules for missing data handling.

Proficiency Model

This section lists all of the Student Models (there is usually one, but there could be more than one) and provides links to the XML and HTML descriptions of those models (gm files). It also gives a table of the names of the available reporting variables. (Note: due to a known bug in StatShop, the reporting variables are sometimes not declared when saving a calibrated model. This is harmless and will be fixed in a future release.)

This section also lists all the available Reporting Rules, scores which can be calculated for a given student. When scoring using this assessment description, scores with the given names will be calculated for each student. (Note: due to a known bug in StatShop, the reporting rules are not saved after calibrating a model.)

Evidence/Link Model Sets

Next come the Evidence and Link Models, organized by task. Depending on the type of Assessment Model Description, it may reference Evidence Model (general rules for scoring tasks from a given Task Model), Link Models (specific rules for scoring a particular task), or Link Model Priors (an in between level between Evidence and Link Models which are associated with Task Specifications). The Evidence and Link Models differ mainly in their level of specificity, they have the same structure and are described with the same kind of gm file. Link Models are always derived from an Evidence Model and have an identical graphical structure. They differ from their Evidence Model ancestor only in the value of parameters.

Whether an Assessment Model Description contains Link Models or Evidence Models depends on the use and origin of the model. Initial model descriptions coming from Portal and intended as input to the StatShop calibrator contain generic Evidence Models. Models output from the calibrator contain task specific Link Models.

Each task can be scored with one or more Link Models. Similarly, each task model can be associated with one or more Evidence Models. In the case of multiples, all of the models are run (for example, there may be one model for overall score and one for subscores). The description presents the Link and Evidence Models sorted by task or task model.

For each Evidence or Link Model, there is a link to the XML gm which contains the appropriate graph and parameters, as well as its HTML human readable version. There is a link to the Student Model used by this link model and a description of the observables. The table gives the external and internal name of the observable as well as the legal values.

In order to promote greater reuse of Evidence Models, Portal contains a facility called a Mapping Rule which maps the name of an observed outcome variable to the name of a node in a graphical model. Thus, the external and internal name could be different. The column marked EID name is the name of the node as output from the Evidence Identification process; in other words, this is the name that StatShop will look for when scoring for this observable. The column marked Node contains StatShop's internal name, as found in the gm file.

Exception Handling Rules

The last part of the table gives rules for handling various kinds of missing data. The table lists the type of exception, the code used in the calibration data and the checksum, as well as the strategies for handling the problem. Types of missing data are:

NA
Outcome is judged to be not-applicable (observable corresponds to a solution path not chosen by the examinee).
Omitted
Examinee saw the task, but did not provide a response to the specified observable. This happens in ICT due to a timeout during the task, or the examinee choosing to go on to the next task before completing the current one.
Not Reached
The task was scheduled to be presented to the examinee, but the examinee never saw it.
Invalid
An unexpected value is included in the data file.

The strategies for handling these exceptions are:

Ignore
Do not score this observable.
Low
Treat this observable as the lowest possible value.
High
Treat this observable as the highest possible value (included for symmetry).
Error
Signal an error and do no process data file.
Warning
Signal a warning, and then treat the value like ignore.

Graphical Model

A graphical model can by described by the collection of variables, and the collection of distributions over those variables. This forms a directed hypergraph, with the variables being the nodes in the graph, and the distributions being directed hyperedges (conditioning variables are from nodes, consequence variable is to node). Another way to draw this construct (as distributions are currently restricted to have one output variable) is to draw an edge from each conditioning variable to the consequence variable. This gives the normal simple graph view.

Graphical models are identified by an ECD ID (Student Model ID or Link ID depending on the type of model) which describes the Portal Project and CAF from which they came, as well as, for Link Models the Task which is associated with them. The syntax for these IDs is meant to mimic a URL with the first part specifying the project, and the last part specifying a unique identifier for the task. Models and submodels appear as directories. in the document

Variables

The first part of the graphical model description is a table of all of the variables included in the model. The Name column gives the internal name of the variable, the External Name gives the name expected from Evidence Identification (if a mapping rule has been defined for this variable). The Role column is used in Evidence/Link Models and may take on one of three values:

SM
A variable "borrowed" from a student model.
Observable
A variable corresponding to an an observable outcome from Evidence Identification.
EM
A latent variable local to the Evidence/Link model, e.g., a context effect.

All variables in StatShop are treated as if they are ordered with the Highest possible value listed first. (Due to a bug in StatShop, the ordered flag on the variable is not always correctly propagated and hence sometimes does not show in the table. This does not affect calculations.)

Distributions

A distribution contains a number of "parent" variable and a single child variable. This is given in a title (parent1,parent2) -> child. This defines the conditional distribution for child given the values of parents. In some cases, the list of parents is empty in which case the distribution is the unconditional distribution of the child. There are many types of distributions which can be divided into two broad super-classes:

Hyper-Dirichlet
Distributions based on specifying the probability table directly, these include the Hyper-Dirichlet, Fixed Distribution, and Fill-in Distribution.
DiBello--Samejima
A reduced parameter class of distributions devised by Lou DiBello based on Samejima's graded response model. These include: Compensatory, Conjunctive, Disjunctive, Inhibitor and Correlation distributions.
The type of the distribution defines its parameters. These parameters are internally translated to a probability table. If present in the XML file, the probability table is shown. (The probably table may have been deleted if the XML file was modified by hand. StatShop works from the parameters and recreates the probability table, thus the parameters always have precedence.) Each row corresponds to a configuration of the parent variables and provides the probability for the child variable given that configuration.

Hyper-Dirichlet Distributions

The natural parameterization of the Bayes net is the Conditional Multinomial Distribution (as in the probability table described above), where each row is an independent multinomial distribution. The natural conjugate prior is the hyper-Dirichlet distribution, in which each row is an independent Dirichlet distribution. The parameter table is given in one of two formats. In the first format, the cell entries for each row are the natural parameters in the Dirichlet model. In the second format, two additional columns: Sum and Scale are included. The effective Dirichlet parameter is computed by multiplying each value in the row by Scale/Sum.

Two other simple distribution types are lumped into this general category. This first is the Fixed Distribution which has no parameters. It is used to define logical relationships and other models whose parameters are not to be learned. It has the same syntax as the hyper-Dirichlet distribution. The second is the Fill-in distribution. These distributions have no child variables and are added to the model to account for dependencies induced by evidence models. Fill-in distributions are equivalent to uniform distributions (unit potentials) and have affect on the graph structure but not the probability distribution.

DiBello--Samejima Distributions

The DiBello--Samejima models are essentially compensatory 2PL versions of Samejima's graded response model, with High, Medium and Low values of the parent variable mapped to thetas of 1,0, and -1 (or +0.67 and -0.67 for the two level context effect). Complete descriptions of the parameterization can be found in Chapter 8 of Bayes Nets in Educational Assessment (currently being written). Here are how the parameters are labeled:

Task Difficulty
Average intercept parameter for all of the Samejima Curve
Level Difficulty Increment
Gap between the curves. This parameter is indexed by the name of the highest level (there is only one in models with only three levels for the observable outcome variable, which covers most of ICT). Note this parameter is restricted to be positive.
Relative Importance
The log of the slope for a variable. This parameter is indexed by the name of one of the parent variables.
Residual Standard Error
Only used in correlation distributions (which are not currently used, but were used in older models). This is the size of the variance (log scale) around the regression line.
Degrees of Freedom
Reserved for future use. This is meant to describe how tightly link models cluster around evidence models in ECD designs, but it is not yet implemented.

The prior distribution for these parameters are normal with the given means and variances expressed through the covariance matrix below the table giving the means. Currently the covariance matrix is always diagonal, although that restriction may be lifted in the future. The prior for the Difficulty Increment parameters is truncated so that the value is always positive.

The various types of the DiBello--Samejima family differ in how they combine the influence of the parent variables. The methods are summarized in the table below:

Compensatory
The effective theta for the distribution is the sum of the effective thetas (times their slope parameters) for the parent variables. The result is scaled by 1/sqrt(number of parents) to stabilize the variance.
Conjunctive
The effective theta for the distribution is the minimum of the effective thetas (times their slope parameters) for the parent variables.
Disjunctive
The effective theta for the distribution is the maximum of the effective thetas (times their slope parameters) for the parent variables.
Inhibitor
One variable acts as a gatekeeper. Unless that parent variable equals or exceeds the threshold, the effective theta is the lowest possible value of the other parent.
Correlation
This model uses cutpoints of the normal distribution to translate between effective theta and probability space, rather than Samejima's graded response model. It is like a regression model with the Residual Standard Error parameter defining the spread around the regression line.
